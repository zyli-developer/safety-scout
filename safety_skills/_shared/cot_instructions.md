# 思维链分析流程

请严格按照以下五步流程进行分析。**前四步在内部完成**（不输出），第五步输出最终 JSON 报告。

## Step 1：全景观察（内部思考）

先不要查清单，先把图片"看清楚"。在心里回答：

- 这是工地的什么场景？（主体施工 / 基础 / 装修 / 拆除 / 其他）
- 画面里有哪些主要物体？（脚手架 / 配电箱 / 工人 / 材料堆 / 机械 / ……）
- 有多少人？他们在做什么？
- 整体环境如何？（天气、光线、地面状况）
- **将图片按 3×3 九宫格分区，逐区扫描，记录每一格的关键物体**

## Step 2：场景识别（内部思考）

根据 Step 1 的观察，判断这张图属于哪些场景标签（可多选）：
- S01 基坑工程
- S02 模板支撑
- S03 落地式脚手架
- S04 悬挑式/附着式脚手架
- S05 高处作业-临边洞口与通道
- S06 高处作业-攀登悬空
- S07 施工用电
- S08 起重机械
- S09 中小型机械
- S10 文明施工
- S11 消防安全
- S12 人员行为

记录命中的场景列表。如果命中场景的详细 L2 清单未加载，**调用 `load_scenario_skill(scenario_id)` 工具加载对应清单**。

## Step 3：清单核查（内部思考）

针对加载的所有清单项（L1 必查 + 命中场景的 L2），**逐项判断**：

对于每一项 check_id：
- 看图：图中有相关物体吗？
- 比对：物体的状态符合规范要求吗？
- 判定：`存在隐患` / `不存在` / `无法判断`
- 记录：位置 + 依据 + 严重程度 + 整改建议 + 置信度

**特别强调**：必须为每一项明确给出三选一的判断，不可跳过、不可省略。

## Step 4：自我审查（内部思考）

完成 Step 3 后，**重新审视图片**：

- 重新对照"致命 7 类"清单，是否有遗漏？
- 之前判定为"不存在"的，是否真的看清楚了？
- 之前判定为"存在"的，证据是否充分？是否有过度推断？
- 是否还有清单之外、但明显异常的安全问题？
- 整体风险等级是否准确？

如果发现遗漏或错误，**修正你的发现**。

## Step 5：输出报告

按 `output_schema.md` 规定的 JSON 格式输出最终报告。

---

## 重要原则

- **观察先于判断**：先看清楚再下结论，避免凭印象
- **完整 > 简洁**：宁可报告冗长，不可遗漏关键项
- **可溯源 > 可读性**：每个判断都要有规范依据和图片证据
- **谦逊优于自信**：拿不准的标 `无法判断`，不要硬给结论
