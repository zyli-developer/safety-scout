# 隐患分类体系

> 上游设计稿：[`../plans/2026-05-18-初始化文档-design.md`](../plans/2026-05-18-初始化文档-design.md) §4

## 来源

分类骨架取自住建部《建筑施工安全检查标准》对**八大伤害**（高处坠落、物体打击、触电、坍塌、机械伤害、火灾、中毒/窒息、起重伤害）的事故类型划分，再补充两个**巡检视角**的工程实用类别：

- `H9 个人防护缺失` —— 见 [§ 关于 H9 的声明](#关于-h9-的声明)
- `H10 其他 / 文明施工` —— 收容不属于八大伤害但常见的现场不规范现象

## H1–H10 类别表

| 编号 | 类别 | 典型现象 |
| --- | --- | --- |
| H1 | 高处坠落 | 临边洞口无防护、脚手架缺挡脚板、安全带未挂、操作平台无护栏 |
| H2 | 物体打击 | 上方堆料、未戴安全帽、起重区下方有人、高处工具未拴挂 |
| H3 | 触电 | 配电箱无门、线路裸露泡水、临电不规范、电缆穿越通道未保护 |
| H4 | 坍塌 | 基坑无支护、模板支撑不稳、堆载过限、土方边坡过陡 |
| H5 | 机械伤害 | 旋转部件无防护罩、维修未停机挂牌、刀具裸露、传送带无急停 |
| H6 | 火灾 | 易燃材料堆放无隔离、动火无看护、消防器材缺失/失效、电气线路超载 |
| H7 | 中毒 / 窒息 | 受限空间未通风、有毒材料无标识、防毒面具缺失、油漆/化学品违规存放 |
| H8 | 起重伤害 | 吊装区无警戒、吊具变形、信号不清、吊钩防脱保险缺失 |
| H9 | 个人防护缺失 | 未戴安全帽 / 反光衣 / 安全带 / 绝缘鞋 / 防护眼镜 / 防尘口罩 / 防护手套；安全帽未系下颌带 |
| H10 | 其他 / 文明施工 | 通道堵塞、料具乱堆、警示标识缺失、临边围挡破损、施工垃圾未清 |

## 每类的字段定义

每个类别在内部数据结构中包含以下字段（用于 LLM Prompt 拼装、报告渲染、未来扩展）：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `code` | string | 是 | `H1` … `H10` 之一 |
| `name` | string | 是 | 类别中文名（如"高处坠落"） |
| `typical_phenomena` | string[] | 是 | 典型现象列表，作为 Prompt 示例 / 报告渲染提示 |
| `severity_hint` | `"high" \| "medium" \| "low"` | 是 | 该类别默认风险等级建议，可被单条隐患的实际 severity 覆盖 |
| `reference_regs` | string[] | 否 | 参考规范条款（如 `JGJ 80-2016`），允许为空数组 |

默认 `severity_hint` 建议：H1 / H4 / H3 / H8 / H6 / H7 = **high**；H2 / H5 = **medium**；H9 / H10 = **medium**（PPE 缺失虽轻易整改，但发生事故时直接致命，不宜定为 low）。

## 关于 H9 的声明

H9 在分类学上是**冗余**的——按住建部"事故类型"分类法，"未戴安全帽"本质是**预防 H2 物体打击**的措施缺失；"未挂安全带"是预防 H1 的缺失；"未穿绝缘鞋"是预防 H3 的缺失。真出事时它们会被记成 H1/H2/H3，不会有独立的"PPE 事故"。

但本系统仍**保留 H9 独立成类**，理由如下：

1. **LLM 友好**：模型看到没戴帽直接打 H9，不必先做"这是 H2 的预防缺失"的二段推理，JSON 输出更稳、token 更省、不容易跑偏到错误事故类别。
2. **巡检视角对齐**：PPE 是工地巡检中**频率最高、检查最直观、整改最便宜**的一类，安全员检查表里 PPE 通常单独成栏。
3. **报告渲染好处**：PPE 类隐患可单独高亮（"现场 N 人未戴安全帽"是高频且醒目的输出）。
4. **数据可聚合**：将来想统计"全工地 PPE 不达标率"直接 group by H9，混入 H1/H2 后得做规则反查。

代价：不严格对齐住建部官方事故类型学。本声明即为这一权衡的存档。

## 供 Prompt 引用的标准化枚举

供 [`prompt.md`](./prompt.md)（Phase 1 PoC 后落定稿）拼装 LLM 输入时直接复制：

```
H1 高处坠落 | H2 物体打击 | H3 触电 | H4 坍塌 | H5 机械伤害 | H6 火灾 | H7 中毒/窒息 | H8 起重伤害 | H9 个人防护缺失 | H10 其他/文明施工
```

LLM 输出的 `category_code` 字段**必须**取自上述枚举，其他取值视为非法响应。
