## Task 1: 写 `docs/specs/hazards.md`

**Files:**
- Create: `docs/specs/hazards.md`

**Source:** 设计稿 §4。

**Step 1: 建目录**

```bash
mkdir -p docs/specs
```

**Step 2: 写文件**

包含 6 节：
1. 顶部 link 引设计稿 §4
2. 开头说明：分类来源 = 住建部「八大伤害」+ 工程实用补充（H9/H10）
3. **H1-H10 表格**（10 行 × 3 列：编号 / 类别 / 典型现象），内容直接抄设计稿
4. 每类字段定义：`code` / `name` / `typical_phenomena` / `severity_hint` / `reference_regs`
5. **关于 H9 的声明**：明确写 "H9 是巡检视角的工程实用类别，不对应住建部官方事故类型"，列出保留理由（LLM 友好 / 巡检视角对齐 / 报告渲染好处 / 数据可聚合）
6. **供 prompt 引用的标准化枚举形式**：

   ```
   H1 高处坠落 | H2 物体打击 | H3 触电 | H4 坍塌 | H5 机械伤害 | H6 火灾 | H7 中毒/窒息 | H8 起重伤害 | H9 个人防护缺失 | H10 其他/文明施工
   ```

**Step 3: 验证**

```bash
test -f docs/specs/hazards.md && grep -c "^| H" docs/specs/hazards.md
```

Expected: 10。

**Step 4: Commit**

```bash
git add docs/specs/hazards.md
```

## Task 2: 写 `docs/specs/report-schema.md`

**Files:**
- Create: `docs/specs/report-schema.md`

**Source:** 设计稿 §6。**前置依赖：** Task 1（schema 中 `category_code` 引用 H1-H10）。

**Step 1: 写文件**

包含：

1. 顶部 link 引设计稿 §6 + `hazards.md`
2. 说明：前后端共享契约，改字段必须同 PR 改本文
3. **完整 JSON 示例**（设计稿 §6 已给）
4. **字段表**（field / type / required / 取值范围 / 说明），重点：
   - `inspection_id`：uuid v4
   - `plain_warning`：1-30 字，口语化，任何角色秒懂
   - `summary`：面向安全员
   - `overall_severity`：高/中/低；定义为 hazards 中最高 severity
   - `hazards[].category_code`：枚举 H1-H10（引 `hazards.md`）
   - `hazards[].severity`：高/中/低
   - `hazards[].regulation`：可空字符串（**不允许**编造条款）
   - `model_meta`：FE 可忽略，BE 必填
5. **变更政策**：任何字段增删 / 取值范围变更视为 breaking change，需同步改 prompt + api + 前端渲染

**Step 2: 验证**

```bash
test -f docs/specs/report-schema.md && grep -c "category_code" docs/specs/report-schema.md
```

Expected: ≥ 2。

**Step 3: Commit**

```bash
git add docs/specs/report-schema.md
git commit -m "docs: 添加报告 JSON schema（specs/report-schema.md）"
```
